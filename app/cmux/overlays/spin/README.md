# SPIN cmux Overlay

This overlay contains SPIN-owned branding/config assets that are applied to a
cmux fork checkout.

Why overlay instead of hand-editing upstream in place:

- Upstream cmux can still be fetched and rebased.
- SPIN-specific product choices are visible and reviewable in this repo.
- Release builds can reproduce the same fork mutation from a clean checkout.

Current overlay scope:

- App identity metadata is generated by `scripts/apply-cmux-spin-overlay.sh`.
- Default SPIN cmux config and Dock controls come from `app/cmux/config/`.
- The SPIN sidebar comes from `app/cmux/sidebars/spin-navigator.swift`.
- Branding source art comes from `assets/branding/spin-icon.svg`.

The long-term cmux fork can eventually absorb these files directly. Keep this
overlay until upstream merge/rebase cadence is proven.
