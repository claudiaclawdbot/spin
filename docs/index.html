01The plant around your agent CLIs

A coding agent harness — primarily omp, with direct CLIs as backup — is a brilliant employee: one session, one folder. SPIN is the org chart, inbox, approval chain, and time clock around it: it schedules, routes, budgets, gates, and audits the work.

🌀

One Navigator loop

Claims a lock file at startup — a second launch just exits, instead of silently doubling your LLM spend.

🧠

Thinks only on change

The brain runs only when watched inputs actually move. An idle org costs a couple of LLM calls a day.

🗂️

State via a CLI, never by hand

Agents call org queue-job …; verbs validate, lock, and append so a mistyped bracket can't corrupt the queue.

🚦

Four hard gates

Acts freely on local, reversible work. Stops for external sends, money, prod deploys, protected pushes.

🔁

Auto-failover

OMP uses configured model roles and fallback chains across your signed-in providers. If OMP itself fails, SPIN falls back to direct CLIs.

🧾

Receipts for everything

Every brain run and job writes an append-only audit trail you can cat.

02cmux is your GUI

SPIN isn't a CLI you babysit — its window is cmux. One command opens the interface: a Coordinator floor you talk to like a person, one sidebar tab per project (each a live omp agent), and the driver coordinating in the background. No Electron, no separate app to maintain — the terminal workspace is the product.

your first minute 
$ spin init            # wizard: providers, OpenRouter, your first project
$ spin up              # opens the cmux interface
  ✓ Coordinator floor open — talk to it like a person
  ✓ driver supervised · live boards up

…then just tell the Coordinator what you want, and it spins up the project:

you  "start a project for my fidget-spinner shop"
└▶ spin new-project fidget-shop ▶ a new fidget-shop tab appears — its own agent, already briefed
🎛️

Coordinator floor

An omp agent you chat with. It onboards you, creates projects, and routes work between them.

🗂️

A tab per project

Each project is a cmux workspace in the sidebar — its own omp orchestrator, its own live board.

♾️

Stays up on its own

spin service install supervises the driver (launchd/systemd) — it respawns on crash, pane-close, or wake.

03Install in 30 seconds

The one-liner sets up everything — clones SPIN, seeds the org files, and installs any missing dependencies for you (node, omp, cmux, an agent CLI). Review it first, like any curl | bash. (Prefer a single offline file? Grab spin-offline.sh — it embeds the whole kit.)

one-liner 
$ curl -fsSL https://raw.githubusercontent.com/claudiaclawdbot/spin/main/spin-bootstrap.sh | bash
or clone 
$ git clone https://github.com/claudiaclawdbot/spin.git ~/spin
$ cd ~/spin && ./install.sh
  ✓ linked: spin, org → ~/.local/bin/
$ spin init            # onboarding wizard → opens the cmux interface

Needs bash, node, and one agent CLI to run headless. omp (oh-my-pi) is the interactive backbone SPIN is built on — every floor agent is an omp session, and it's the Pi in SPIN; cmux is the optional visual layer. The installer sets all of this up.

04SPIN.app for Mac

SPIN.app is the Mac product path: a self-contained app with SPIN-branded cmux as the native workspace UI, bundled OMP/Pi as the agent engine, and the same auditable plain-file SPIN runtime inside the app. The CLI remains available for power users, automation, debugging, and headless recovery.

🖥️

Self-contained bundle

SPIN.app carries its cmux-compatible UI app, cmux, omp, spin-agent, runtime scripts, icon, dock controls, and notices.

🧪

Open-source DMG

The public Mac distribution path is the GitHub beta DMG. It is ad-hoc signed and not notarized, so checksum verification and right-click Open are documented.

🔏

No App Store wait

SPIN is not waiting on the Mac App Store or Apple Developer ID. Developer ID notarization can reduce Gatekeeper friction later, but it is not required for this app path.

build the checked beta DMG 
$ SPIN_RELEASE_FORMAT=dmg scripts/release-macos.sh --source-cmux
$ scripts/prepare-open-source-release.sh --artifact dist/release/SPIN-*-macos-*.dmg
  ✓ dmg · sha256 · manifest · open-source tester notes

When a beta DMG is attached to a GitHub release, install by verifying the SHA-256, opening the DMG, dragging SPIN.app onto Applications, and opening it. If macOS warns because the build is not notarized, use right-click Open after verifying the checksum. Full instructions live in macOS beta install guide.

05It's just files
