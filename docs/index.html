01One workspace for every project

SPIN brings project agents, execution state, model routing, and approval controls into one Mac app without mixing unrelated repository contexts.

Portfolio view

See priorities, active work, blockers, and approvals across every registered project.

Isolated workspaces

Each project keeps its own repository, agent context, queue, board, and receipt history.

Persistent execution

Supervised background jobs continue independently of a visible terminal pane.

Consistent delegation

Every handoff includes the objective, paths, constraints, checks, and reporting requirements.

Flexible model routing

OMP manages authenticated providers, model roles, retries, and fallback chains.

Verifiable results

Queues, handoffs, decisions, and receipts remain inspectable in local files.

02Project isolation comes first

SPIN coordinates separate project agents instead of forcing an entire portfolio into one shared prompt. Model and provider portability sits underneath that project layer.

Across projects

Agents cooperate through durable queues, handoffs, receipts, and Navigator decisions while retaining focused repository context.

Across models

OMP can route work through configured OpenAI, Anthropic, OpenRouter, Gemini, or local provider lanes.

Across sessions

File-backed state and supervised processes preserve work through terminal changes and app relaunches.

03A clear operating model

The Navigator owns cross-project priorities and routing. Project agents own implementation inside their registered repositories. The background driver keeps queued work moving.

workspace status
$ spin up
  [ok] Navigator online
  [ok] Project workspaces restored
  [ok] Background driver supervised

Navigator

Creates projects, routes work, refines handoffs, tracks blockers, and surfaces approvals.

Project agents

Execute scoped work with the repository context, tools, model policy, and checks for one project.

Background driver

Dispatches durable jobs, monitors state changes, records outcomes, and prevents duplicate cycles.

04SPIN for Mac

The Mac app packages the SPIN-branded workspace, OMP/Pi agent runtime, project orchestration, health checks, and update controls in one installation.

Self-contained app

Includes the workspace engine, omp, spin-agent, runtime scripts, health checks, and product assets.

Provider setup through OMP

Provider credentials remain in OMP or the provider's standard CLI configuration.

Local runtime state

Project metadata, queues, approvals, and receipts stay under Application Support on the Mac.

Checked updates

Downloaded releases are verified before app code is replaced, with backup and rollback metadata.

05Source and CLI

The source install supports Linux, headless operation, automation, debugging, and app development. The standard Mac install does not require it.

clone and install
$ git clone https://github.com/claudiaclawdbot/spin.git ~/spin
$ cd ~/spin && ./install.sh
$ spin init
$ spin

The source lane requires bash, node, and OMP or a supported fallback agent CLI. Add cmux for the visual workspace outside the packaged app.

06Local and inspectable
