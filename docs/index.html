01The plant around your agent CLIs

Coding-agent CLIs (Claude Code, Codex, Gemini, omp) are brilliant employees — one session, one folder. SPIN is the org chart, inbox, approval chain, and time clock around them: it schedules, routes, budgets, gates, and audits the work.

🌀

One Navigator loop

Claims a lock file at startup — a second launch just exits, instead of silently doubling your LLM spend.

🧠

Thinks only on change

The brain runs only when watched inputs actually move. An idle org costs a couple of LLM calls a day.

🗂️

State via a CLI, never by hand

Agents call org queue-job …; verbs validate, lock, and append so a mistyped bracket can't corrupt the queue.

🚦

Four hard gates

Acts freely on local, reversible work. Stops for external sends, money, prod deploys, protected pushes.

🔁

Auto-failover

codex → claude → gemini → ollama. A rate-limited CLI is benched; the waterfall advances. No dead-ends.

🧾

Receipts for everything

Every brain run and job writes an append-only audit trail you can cat.

02Install in 30 seconds

One self-extracting file packages the whole setup — scripts, prompts, org skeleton — and installs any missing dependencies for you (node, omp, cmux, an agent CLI). Review it first, like any curl | bash.

one-liner 
$ curl -fsSL https://raw.githubusercontent.com/claudiaclawdbot/spin/main/spin-bootstrap.sh | bash
or clone 
$ git clone https://github.com/claudiaclawdbot/spin.git ~/spin
$ cd ~/spin && ./install.sh
  ✓ linked: spin, org → ~/.local/bin/
$ spin doctor          # checks deps, driver, floors
$ spin start           # launch the Navigator loop

Needs bash, node, and one agent CLI to run headless. omp (oh-my-pi) is the interactive backbone SPIN is built on — every floor agent is an omp session, and it's the Pi in SPIN; cmux is the optional visual layer. The installer sets all of this up.

03It's just files

No database, no message broker, no daemon you can't read. The whole org talks through plain files you can grep and audit.

you ──spin approve──▶ APPROVALS.md ──▶ SPIN Navigator ──▶ AGENT_QUEUE.json ──▶ detached agent jobs
you ◀──spin status── HUMAN_QUEUE.md ◀── tick loop ◀── INBOX.md ◀── project receipts

04The four gates

SPIN does local, reversible work without asking. It stops and waits for exactly four things — enforced behaviorally by every prompt, surfaced as an approval queue.

📤
External sendsemail, DM, form, public post
💸
Spending moneygas, wallets, paid APIs
🚀
Production deploysanything that ships to prod
🔒
Protected pushesmain or any human-owned repo
